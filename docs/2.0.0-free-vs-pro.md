# FooConvert 2.0.0 Release Breakdown

This document summarizes the major customer-facing changes on `feature/pivot` compared with the previous `origin/master` release baseline in both the free plugin and the bundled `pro` package.

## Moved to Free

### 1. Professional templates and font support

**Marketing summary**

Free users can now start from polished, conversion-ready popup, bar, and flyout designs instead of building everything from scratch. The supporting Google Font workflow also moves down into free, so those campaigns can stay on-brand without a Pro upgrade.

**Changelog-ready**

- Moved 5 professionally designed popup, bar, and flyout templates from Pro to free.
- Moved Google Fonts support and font settings from Pro to free.
- Registered the bundled template fonts in the free plugin.

### 2. Sign Up, Countdown, and Coupon blocks

**Marketing summary**

The free editor now includes the core conversion blocks needed for lead capture, urgency campaigns, and promo messaging, which makes the free version far more complete out of the box.

**Changelog-ready**

- Moved the Sign Up block from Pro to free.
- Moved the Countdown block from Pro to free.
- Moved the Coupon block from Pro to free.

### 3. Leads management in wp-admin

**Marketing summary**

Free users can now capture, review, filter, and export leads directly inside WordPress. That turns FooConvert free into a practical lead-generation workflow, not just a popup builder.

**Changelog-ready**

- Moved the Leads admin screen from Pro to free.
- Added free lead filtering by date range.
- Added free lead export from the leads table.

## Introduced in Pro

### 1. WooCommerce targeting and trigger automation

**Marketing summary**

Pro can now react to real storefront intent, so merchants can target campaigns around product interest, cart behavior, coupon actions, and checkout friction instead of relying only on generic triggers.

**Changelog-ready**

- Added WooCommerce display rules for shop, cart, checkout, account, product, product tag, product category, and WooCommerce template pages.
- Added WooCommerce triggers for cart add, cart update, cart remove, coupon applied, invalid coupon, cart idle, checkout enter, checkout exit, checkout error, payment failed, product viewed, and product high intent events.
- Added product and subtotal conditions for WooCommerce trigger configuration.

### 2. New Pro conversion blocks and effects

**Marketing summary**

Pro now ships with store-focused conversion blocks that help visitors redeem offers, track progress toward incentives, and add more momentum to key campaigns.

**Changelog-ready**

- Added the Apply Coupon block.
- Added the Free Shipping Progress block.
- Added the Free Shipping Text block.
- Added the Free Shipping Bar block.
- Added the Cart Threshold Progress block.
- Added the Confetti block.

### 3. Sales attribution and revenue reporting

**Marketing summary**

Pro users can now measure popup performance in revenue, not just clicks and views. Sales attribution makes it easier to identify which campaigns actually drive orders.

**Changelog-ready**

- Added WooCommerce sale attribution tracking.
- Added sales totals and sales count metrics to popup stats.
- Added Recent Sales and Totals By Popup dashboard panels.
- Added revenue-aware top performer metrics and sales datasets to reporting.

### 4. A/B testing experiments

**Marketing summary**

Pro now includes built-in popup experimentation, so marketers can run tests, compare variants, track confidence, and let automation finish or apply winners once enough data is available.

**Changelog-ready**

- Added A/B testing experiments for popups.
- Added experiment dashboards and editor status integrations.
- Added automation for auto-winner completion and apply-winner flows.
- Added an option to email the site admin when experiment automation completes or applies a winner.

## Other Core Workflow Updates

### 1. Unified popup management

**Marketing summary**

The admin experience is simpler because popups, bars, and flyouts now live inside one Popup workflow with a type chooser, shared stats, and a cleaner list screen.

**Changelog-ready**

- Unified popups, bars, and flyouts under a single Popup post type.
- Added a popup type chooser before opening the editor.
- Added popup type columns and filters in the popup list table.
- Updated admin terminology from widgets to popups.

### 2. Faster display rule management

**Marketing summary**

Users can now review and update targeting from the main Popups screen, which removes needless trips into the editor for small rule changes.

**Changelog-ready**

- Added a Display Rules column to the popup list table.
- Added inline display rule summaries and modal editing from the popup list screen.
- Made display-rule admin locking extensible for Pro experiment workflows.
