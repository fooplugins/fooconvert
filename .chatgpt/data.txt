This file outlines how data is stored within FooConvert.

FooConvert stores all popup types in a single WordPress custom post type:
- `fc-popup`

The logical popup type is stored in post meta:
- `_fooconvert_popup_type`
- canonical values: `bar`, `flyout`, `overlay`
- legacy `popup` is accepted only during normalization and migration

Each popup stores its editor content in the standard WordPress `post_content` field for its post entry.
That content contains the popup root block plus any nested blocks used inside the popup.
The root block is one of:
- `fc/bar`
- `fc/flyout`
- `fc/overlay`

The free plugin now owns these content blocks:
- `fc/countdown`
- `fc/coupon`
- `fc/sign-up`

The free plugin also owns the migrated former PRO templates, template media, template fonts, and basic lead-management UI.

Saved popup content may contain baked-in asset URLs or legacy overlay block names.
FooConvert includes one-time content migrations that rewrite:
- stale media URLs from `plugins/fooconvert/pro/assets/media/` to `plugins/fooconvert/assets/media/`
- legacy overlay content from `popup` / `fc/popup*` to `overlay` / `fc/overlay*`

Migration completion is recorded in the WordPress option:
- `fooconvert-content-migrations`

FooConvert creates a custom database table for analytics events:
- `fooconvert_events`

Its purpose is to store popup analytics and engagement/conversion event data.

The SQL shape is:

CREATE TABLE fooconvert_events (
    id bigint(20) unsigned NOT NULL AUTO_INCREMENT,
    widget_id bigint(20) unsigned NOT NULL,
    event_type varchar(255) NOT NULL,
    event_subtype varchar(255) DEFAULT NULL,
    conversion tinyint(1) DEFAULT NULL,
    sentiment tinyint(1) DEFAULT NULL,
    page_url text DEFAULT NULL,
    device_type varchar(50) DEFAULT NULL,
    anonymous_user_guid varchar(255) DEFAULT NULL,
    user_id bigint(20) unsigned DEFAULT NULL,
    extra_data longtext DEFAULT NULL,
    timestamp datetime DEFAULT $timestamp_default,
    PRIMARY KEY (id)
);

Field summary:
- `id`: primary key
- `widget_id`: popup post ID associated with the event
- `event_type`: core event type such as `open`, `click`, `close`, or `update`
- `event_subtype`: subtype such as `engagement`, or `NULL`
- `conversion`: whether the event counts as a conversion
- `sentiment`: positive/negative/neutral sentiment marker
- `page_url`: URL where the event occurred
- `device_type`: device classification for the event
- `user_id`: logged-in WordPress user ID, if available
- `anonymous_user_guid`: anonymous frontend visitor identifier when no user is logged in
- `extra_data`: extra event payload such as conversion metadata
- `timestamp`: when the event occurred

Lead capture writes through free plugin code.
Basic lead viewing and export are now also owned by the free plugin.
PRO keeps only lead integrations and advanced premium lead/analytics extensions.

There is a default analytics retention period of 14 days.
Older event data is removed by a scheduled cron cleanup, while PRO can extend the retention behavior.
